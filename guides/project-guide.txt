# Team Guide — Weather for Construction

## What this project is
A React-based weather dashboard designed for outdoor construction workers.
It shows key information quickly: temperature, rain, wind, warnings, and simple safety recommendations.

---

## Technologies We Use (Simple Explanation)

- **React** – Builds the user interface using components.
- **Vite** – Runs the development server and builds the project.
- **Tailwind CSS** – Used for styling with utility classes.
- **OpenWeather API** – Provides weather data.
- **Browser Geolocation** – Detects the user’s location automatically.

---

## Why We Run `npm install`

When you clone the repository, it does NOT include the `node_modules` folder.

`node_modules` contains all the libraries the project depends on (React, Vite, Tailwind, etc.).

Running:

npm install

- Reads `package.json`
- Downloads all required dependencies
- Recreates the `node_modules` folder

Without running `npm install`, the project will not work.

You only need to run it:
- The first time you clone the project
- If new dependencies are added

---

## How to Run the Project

1. Install dependencies:
   npm install

2. Create a `.env` file in the root (already done):
   VITE_OPENWEATHER_API_KEY=YOUR_KEY_HERE

3. Start the development server:
   npm run dev

⚠️ Do not commit `.env` (it contains the API key).

## How We Will Build the App (Simple Task Breakdown)

We will build the app step-by-step. Each step is small and can be done by one person.

### Step 1 — Basic Layout (UI only)
Goal: Show a simple dashboard screen

Tasks:
- Create a main layout in `App.jsx`
- Add:
  - Location text (placeholder)
  - Big temperature text (placeholder)
  - Section for warnings
  - Section for extra info (wind, rain)
- Style using Tailwind

No API yet — just static text.

---

### Step 2 — Get User Location
Goal: Detect user location (GPS)

Tasks:
- Use browser geolocation:
  `navigator.geolocation.getCurrentPosition`
- Store latitude and longitude
- Show error if permission denied

---

### Step 3 — Fetch Weather Data
Goal: Connect to OpenWeather API

Tasks:
- Create a function in `src/api/openWeather.js`
- Use `fetch` to get weather using lat/lon
- Store result in state
- Show:
  - Temperature
  - Weather condition

---

### Step 4 — Display Real Data
Goal: Replace placeholders with real data

Tasks:
- Show real temperature
- Show wind speed
- Show rain/precipitation
- Show location name

---

### Step 5 — Add Warnings (Important Feature)
Goal: Make app useful for construction workers

Tasks:
- Create simple rules:
  - High wind → warning
  - Heavy rain → warning
  - High temperature → warning
- Display warning banner

---

### Step 6 — Add Recommendations
Goal: Help users make decisions

Tasks:
- Based on weather:
  - “Strong winds — avoid working at height”
  - “Heavy rain — slippery surfaces”
  - “Hot weather — take breaks”
- Show message on screen

---

### Step 7 — Improve UI
Goal: Make it clean and easy to read

Tasks:
- Make temperature large and centered
- Make warnings stand out (color)
- Keep everything on one screen
- Make it readable outdoors

---

### Step 8 — Final Checks
Goal: Make sure everything works

Tasks:
- Test on different screen sizes
- Test when location is denied
- Test when API fails
- Fix bugs

---

## Important Rule

Always keep the app:
- Simple
- One screen
- Fast to read
- Focused on key information only

Do NOT:
- Add unnecessary features
- Overcomplicate the UI
